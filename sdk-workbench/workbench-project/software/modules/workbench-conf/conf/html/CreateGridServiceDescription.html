Create Grid Service

The caBIG program is aimed at creating a data sharing network for the benefit of the cancer research community. In the caBIG, the data sharing network infrastructure is known as caGrid. The caGrid provides a rich set of tools to create and manage lifecycle of a grid service. Primary tool for creating a grid service is known as "Introduce". Introduce tool is capable of creating two types of grid service:

-                 Data Service is a grid service which provides access to the data source using a standard Query By Example (QBE) interface. The data service portion of Introduce can seamlessly integrate with the caCORE SDK generated system (i.e. application generated by the Create Application workflow). Developers can further develop a custom data service to connect to a non-SDK generated system.

-                 Analytical Service is a grid service which provides access to the underlying functionality via a user defined interface. Custom operations can be for a scientific algorithm to crunch datasets or any other functionality not provided by the standard data service interface.

Introduce Tool

Introduce is an open-source, extensible toolkit designed to support the development and deployment of WS/WSRF-compliant Grid services. Introduce aims to reduce the effort required to develop and deploy Grid services by hiding low-level details of the Globus Toolkit from service developers, and to enable the implementation of strongly-typed Grid services.

The key features of Introduce include:

-         A graphical user interface (GUI) that allows service developers to easily create and modify Grid service interfaces. Developers can also write their own service description documents, which the Introduce toolkit can use to create and modify a service programmatically.

-         High-level functions that encapsulate and hide the common complexities and low-level command-line tools of the Globus Toolkit. These include client and server wrappers to encapsulate the "boxed" document literal Grid service calls, functions to create configurable service properties, functions to specify resource properties and register metadata, functions to specify the security configurations of the service operations of the service, and support for deploying a service to commonly used Grid service containers (such as JBoss or Tomcat).

-         Support for the development of strongly-typed services. Using plug-ins, Introduce enables discovery of datatypes from virtually any style of datatype provider.

-         Extension Plug-ins, which allow Introduce to be customized and its base functionality to be extended in two ways: 1) For custom and common service types in an application domain and 2) To employ customized discovery mechanisms for common datatypes for creating strongly-typed services.

-         Support for the implementation of secure services. Introduce leverages all aspects of the GSI in order to provide customizable service-level and method-level security configuration and code generation. It provides support for service developers to optionally turn on authentication and authorization support for individual service methods as well as for the entire service itself.

-         Management of the service-specific files and directories required by the Globus Toolkit for correct compilation and deployment. Introduce also generates appropriate, object-oriented client APIs, which can be used by client applications to interact with the service.

Introduce Output

When you are finished with the Introduce process, the output you should end up with includes:

-         <Project-name> folder containing the service and the client

Once you have that, you can deploy the project into the "wsrf" folder on an application server like JBoss or Tomcat using Introduce. Some people refer to this as Introduce "output".

Then, when the Introduce-generated service starts running, it sends "heartbeats" to the caGrid Index Service. That's how the Index service knows how many services are alive. The caGrid Portal then queries the Index Service to show to the world what is available on the Grid.
