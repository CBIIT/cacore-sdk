Model Mapping

Model mapping is the process of linking Logical Model of the system with the Data Model. This step is required by the caCORE SDK to generate a caCORE-like system from a UML model. If the developer intends to create any other type of system then this workflow can be ignored.

The specific mapping between the Logical Model and the Data Model is done by adding additional information to the UML Model using UML Tagged Values. This task can be accomplished by one of the two tools:


-         UML Modeling tool (i.e. Enterprise Architect or ArgoUML)

-         caAdapter MMS (recommended)

UML Modeling Tool

Developers can use a UML modeling tool to specify the mapping information between the logical and the data model. In the UML model, the mapping information can be specified by using the UML Dependency association and Tag Values. Here's a brief outline of the process:


-         Create a new class diagram

-         Add the Data Model class representing the table to the diagram

-         Add the Logical Model class representing the domain object to the diagram

-         Using a UML Dependency association, draw unidirectional link from the table (source) to the domain object (target)

-         To the UML Dependency association link, add a stereotype named 'DataSource'.

-         For each of the table attributes, add a 'mapped-attributes' tag with a value of <fully qualified domain object class name>.<attribute name>.

Once the mapping is complete, you can save the changes, export the newly tagged XMI file from your UML Modeling tool (Enterprise Architect only), and use it in the 'Create Application' workflow. This workflow will generate various artifacts by reading the information from the UML model.

caAdapter MMS

caAdapter Model Mapping Service (MMS) allows users to map an object to a data model. The tool provides a rich user interface with drag and drop capability to perform this mapping. Subsequently, the tool annotates the XMI file containing the object and data models with the proper mapping tags.

Once the mapping is complete, you can save the newly tagged XMI file from caAdapter and use it in the Create application/ code generation workflow. The create application workflow will generate various artifacts by reading the information from the UML model.
